# Define SCSS Variables

- Type: Task
- Priority: High
- Story Points: 3
- Labels: styles, scss, design-system, shared, foundation
- Git Branch: `feat/workbench-25-scss-variables`

## Description

Define comprehensive SCSS variables for color palette, spacing scale, typography, and breakpoints to establish a consistent design system foundation. All styles must use these variables instead of hardcoded values.

## User Story

As a developer, I want centralized SCSS variables for colors, spacing, typography, and breakpoints so that I can maintain consistent styling across the application and easily update the design system.

## Acceptance Criteria

- [ ] **Color Palette Variables**
  - [ ] Define primary color palette (primary, secondary, accent colors)
  - [ ] Define neutral colors (grays, black, white)
  - [ ] Define semantic colors (success, error, warning, info)
  - [ ] Create color variables in `styles/variables/colors.scss`
  - [ ] Use descriptive names (e.g., `$color-primary-500`, `$color-error-600`)

- [ ] **Spacing Scale Variables**
  - [ ] Define spacing scale (margins, paddings, gaps)
  - [ ] Use consistent scale (e.g., 4px, 8px, 12px, 16px, 24px, 32px, 48px, 64px)
  - [ ] Create spacing variables in `styles/variables/spacing.scss`
  - [ ] Use descriptive names (e.g., `$spacing-xs`, `$spacing-sm`, `$spacing-md`, `$spacing-lg`)

- [ ] **Typography Scale Variables**
  - [ ] Define font sizes (xs, sm, base, lg, xl, 2xl, 3xl, etc.)
  - [ ] Define line heights for each font size
  - [ ] Define font weights (light, normal, medium, semibold, bold)
  - [ ] Create typography variables in `styles/variables/typography.scss`
  - [ ] Use descriptive names (e.g., `$font-size-base`, `$line-height-base`)

- [ ] **Breakpoint Variables**
  - [ ] Define responsive breakpoints (mobile, tablet, desktop)
  - [ ] Use consistent breakpoint values (e.g., 640px, 768px, 1024px, 1280px)
  - [ ] Create breakpoint variables in `styles/variables/breakpoints.scss`
  - [ ] Use descriptive names (e.g., `$breakpoint-sm`, `$breakpoint-md`, `$breakpoint-lg`)

- [ ] **Additional Variables**
  - [ ] Define border radius values in `styles/variables/border-radius.scss`
  - [ ] Define shadow values in `styles/variables/shadows.scss`
  - [ ] Define transition durations and easings in `styles/variables/transitions.scss`

- [ ] **Variable Organization**
  - [ ] All variables organized in `styles/variables/` directory
  - [ ] Each variable file exports variables using `@use` and `@forward`
  - [ ] Variables follow naming conventions (kebab-case with descriptive prefixes)
  - [ ] Variables documented with comments where needed

## Technical Considerations

- **Shared Layer** (`styles/variables/`):
  - Create `colors.scss` for color palette
  - Create `spacing.scss` for spacing scale
  - Create `typography.scss` for typography scale
  - Create `breakpoints.scss` for responsive breakpoints
  - Create `border-radius.scss` for border radius values
  - Create `shadows.scss` for shadow values
  - Create `transitions.scss` for transition values
  - All files use SCSS `@use` and `@forward` for proper module system

- **Import Pattern**:
  - Variables should be importable via `@use "@/styles/variables/colors" as *;`
  - Follow SCSS module system best practices
  - Ensure variables are available globally when needed
