# Global Styles

**Type:** Task  
**Priority:** High  
**Story Points:** 3  
**Labels:** setup, styles, shared  
**Git Branch:** `feat/workbench-35-global-styles`

## Description

Setup global SCSS file with CSS reset, base styles, and typography defaults. This provides a consistent foundation for all components and ensures proper browser normalization.

## User Story

As a developer, I want to have global styles with CSS reset and base typography so that all components start from a consistent baseline without browser-specific inconsistencies.

## Acceptance Criteria

1. Global SCSS file created at `styles/global.scss` with CSS reset (normalize or custom reset)
2. Base typography styles applied to body and common HTML elements
3. Base styles for common elements (headings, links, lists, etc.) using SCSS variables
4. Global styles properly imported in the root layout
5. No browser-specific inconsistencies in default styling

## Technical Considerations

- **Shared Layer**: Global styles in `styles/global.scss` applied at application root
- **SCSS Variables**: Use variables from `styles/variables/*` for all values
- **Next.js Integration**: Import in `app/layout.tsx` or appropriate layout file
- **Reset Strategy**: Choose between normalize.css, modern-normalize, or custom reset

## Definition of Done

- [ ] Global SCSS file created with CSS reset
- [ ] Base typography styles applied using typography variables
- [ ] Base element styles (headings, links, lists) using color and spacing variables
- [ ] Global styles imported in root layout
- [ ] No hardcoded values (all use SCSS variables)
- [ ] Cross-browser consistency verified
